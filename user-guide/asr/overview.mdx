Open LLM VTuber supports several ASR (Automatic Speech Recognition) backends. The active backend is set by `asr_model` inside `character_config.asr_config`.

```yaml
character_config:
  asr_config:
    asr_model: 'sherpa_onnx_asr'
```

Change `asr_model` to the key of any backend listed below, then add or update the matching config block.

## Supported backends

| Backend | Type | Languages | Notes |
|---|---|---|---|
| `sherpa_onnx_asr` | Offline | Multi (model-dependent) | Default. Fully offline, no setup required for SenseVoice. |
| `faster_whisper` | Offline | 99+ | High accuracy; requires more VRAM than sherpa_onnx. |
| `whisper_cpp` | Offline | 99+ | CPU-optimized Whisper port. |
| `whisper` | Offline | 99+ | Original OpenAI Whisper. Slowest on CPU. |
| `fun_asr` | Offline\* | zh, en, auto | Downloads model on first launch; can run offline after. |
| `groq_whisper_asr` | Cloud | 99+ | Fast cloud transcription via Groq. |
| `azure_asr` | Cloud | 100+ | Microsoft Azure Speech Service. Multi-language detection. |

\* `fun_asr` requires an internet connection on every launch to check for model updates unless `disable_update: true` is set.

## Recommendations

- **Offline use**: Start with `sherpa_onnx_asr` (default). If you need higher accuracy on non-CJK languages, switch to `faster_whisper`.
- **Cloud use**: Use `groq_whisper_asr` for low-latency transcription at no cost on the free tier. Use `azure_asr` if you need robust multi-language auto-detection.
- **Chinese-focused**: `sherpa_onnx_asr` with `sense_voice` or `fun_asr` both work well for Mandarin.

## Setting asr_model

```yaml
character_config:
  asr_config:
    asr_model: 'sherpa_onnx_asr'  # Change this to switch backends

    sherpa_onnx_asr:
      model_type: 'sense_voice'
      # ... other fields for this backend
```

Only the block matching the active `asr_model` value is loaded. Other blocks are ignored.
