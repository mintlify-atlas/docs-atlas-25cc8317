## Important file locations

| Live2D models | `live2d-models/` |
| Chat history | `chat_history/` |
| TTS audio cache | `cache/` |

## Configuration flow

When the server starts, `run_server.py` reads `conf.yaml` and builds a `Config` object using the Pydantic models in `config_manager/`. The server then creates a single **default** `ServiceContext` from this config. When a new WebSocket client connects, the handler clones the default context so each session has its own engine instances.

<Note>
Character-specific overrides in `characters/` are **deep-merged** with the base `conf.yaml` at runtime when a client switches configurations via the `switch-config` message.
</Note>
